Consent-first local support prototype

DeskRTC

A host shares a screen, every viewer must pass PIN verification, and the host then approves that viewer with an explicit per-session permission scope.

Host this computer Request access to a host

Attended access

PIN verification creates only a short-lived request. The host must separately approve screen viewing and any pointer, keyboard, or clipboard capability.

Local-first transport

The default server binds to loopback and does not use a third-party STUN or TURN service. Internet deployment requires HTTPS, authentication, and reviewed relay infrastructure.

Authorized support only

Do not use DeskRTC for hidden access, surveillance, credential collection, identity-verification evasion, or control without informed host consent.
